Underneath it's all just bits

Bit (Binary digit)

Definition

Direct representation of digital logic values

  • Can be 0 or 1
  • Multiple bits are used to represent complex data items
  • Same underlying hardware can represent bits of an integer or bits of a character

Byte

Definition

Set of multiple bits

  • Size depends on underlying hardware (computer)
    • CDC - 6 bits
    • BB&N - 10 bits
    • IBM - 8 bits (standard nowadays)
  • On most computers the byte is the smallest addressable unit of storage

Byte size

Definition

Numbers of bits per byte determines range of values that can be stored

  • Byte of k bits can store values
    • 6 bits - 64 values
    • 8 bits - 256 values
    • 10 bits - 1024 values

Binary representation

Bits themselves have no meaning
Byte just stores 1's and 0's
i.e:

  • 000 001 010 011
  • 100 101 110 111

All meaning is determined by how the bits are interpreted

Example

Possible interpretation of 3 bits

  • Device status
    • First bit is 1 if disk is connected
    • Seconds bit is 1 if printer is connected
    • Third bit is 1 is keyboard is connected
  • Integer interpretation
    • Positional interpretation uses base 2
    • Values are from 0 to 7
    • Order of bits must be specified

Any system to decimal

  • n - amount of decimals before the coma
  • i - amount of decimals after the coma
  • - number of i-th order
  • p - base of the number system
Decimal to any system
  1. For the whole part:
    1. Divide by the base of the system
    2. Keep track of quotient and remainder
    3. Keep dividing the quotient until you get a 0
    4. Write remainders in reverse order
  2. For the fractional part:
    1. Multiply the fractional part by the base
    2. Take the whole part
      1. 1,345 becomes 0,345, 1 is recorded
      2. 0,765 does not change, 0 is recorded
    3. Repeat until the desired precision is reached
Binary to hex/octal and reverse

Group/ungroup by 4 for hex, by 3 for octal

Unsigned integers

  • Straightforward positional interpretation
  • Each successive bit represents next power of 2
  • No negative numbers (A set of k bits can represent integers from 0 to )
  • Precision is fixed (size of integer is constant)
  • Arithmetic operations can produce overflow/underflow (result can't be represented in k bits)
  • Overflow is handled with wraparound and carry bit

Overflow

  • Values wrap around address space
  • Hardware records overflow in a separate carry indicator
    • Software must check it after operations
    • Can be used to raise an exception

Sign magnitude

  • Familiar to humans
  • First bit represents sign
  • Successive bits represent absolute value of integer
  • ? Quirk: can create -0

One's complement

  • Positive numbers use positional representation
  • Negative number are formed by inverting all bits of positive value
  • ? Quirk: two representations for 0: 0000 and 1111
Note

Some checksum algorthms use one's complement

Two's complement

  • Positive numbers use positional representation
  • Negative numbers formed by subtracting 1 from positive value and inverting all bits
    • 0010 represents 2
    • 1110 represents -2
    • High order bit is set if number is negative
  • ? Quirk: one more negative value

Implementation

  • We consider using unsigned two's complement together because a single piece of hardware can handle arithmetic of both.
  • Software can choose an interpretation for each integer
Example

4 bits

  • Adding 1 to binary 1001 produces 1010
  • Unsigned interpretation goes from 9 to 10
  • Two's complement interpretation goes from -7 to -6

Sign extension

  • Needed for unsigned and two's complement representations
  • Used to accommodate multiple sizes of integers
  • Extends high-order bit (sign bit)

When using two's complement representation and assigning 32-bit integer to 64-bit integer, upper 32 bits are filled with

  • 0 for a positive number
  • 1 for a negative number
  • (High order bit is replicated)
Example
  • The 8-bit version of integer -3 is
    • 11111101
  • The 16-bit version of integer -3 is
    • 1111111111111101

Sign extension also applies to bit-shifting

Example

Shifting -4 one bit right should produce -2 (divide by 2)

  • 16-bit representation of -4 is
    • 1111 1111 1111 1100
  • After right shift, value is -2
    • 1111 1111 1111 1110

So, for right shift we also replicate high order bit

Order of bits and bytes

  • @ Need to choose order for
    • Storage in physical memory system
    • Transmission over a network
  1. Bit order
    • Handled by hardware
    • Usually hidden from programmer
  2. Byte order
    • Affects multi-byte data items (i.e: integers)
    • Visible and important to programmer

Endianness

Little endian

Least significant byte of integer is in lowest memory location

Big Endian

Most significant byte of integer is in lowest memory location

Binary coded decimal (BCD)

  • Pioneered by IBM
  • Represents integer as a string of bytes
    • Unpacked: one digit per 8-bit byte
    • Packed: one digit per 4-bit nibble
  • Uses sign magntude representation
Example

Unpacked BCD

  • 123456 is stored as
    • 0x01 0x02 0x03 0x04 0x05 0x06
  • -123456 is stored as
    • 0x01 0x02 0x03 0x04 0x05 0x06 0x0D
  • ! Disadvantages
    • Takes up more space
    • Hardware is slower than integer or floating point
  • $ Advantages
    • Gives results humans expect
    • Avoids repeating binary value for .01
Note

Preferred by banks
