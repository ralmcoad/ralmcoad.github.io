Tree

Definition

Connected and acyclic graph is called a tree

Example
Note
  1. Any vertex of a tree with a degree is an articulation
  2. Any edge of a tree is a bridge
  3. Any tree with vertices has edges (forest will have less)
  4. Graph is a tree if and only if any 2 vertices can be connected by only 1 circuit

Kruskal’s algorithm

  1. Choose an edge of smallest weight, if it will not make a cycle
  2. Add this edge to the tree, delete it from initial graph
  3. Repeat while possible
  • All edges in ascending order
  • Construct a new tree (green edges)

Edge elimination from a cycle

  1. Find any cycle
  2. Find an edge in this cycle having biggest weight
  3. Eliminate this edge
  4. Continue until no more cycles left

Breadth-first search

  1. Choose a vertex from the graph
  2. Look for all it's adjacent vertices, add them to the n-th level of the tree (if not already)
  3. Now search for adjacent vertices of those vertices
  4. Repeat until all vertices are added

Storing trees

We can store a tree with n vertices in array of elements

Tree to array

Suppose we have a tree:

  1. Enumerate all vertices, get a list of rising vertices is: [1, 2, 3, 7, 8, 9] (marked #c/redred)

  2. Find a rising vertex with smallest index (in our case it's 1). It is connected to vertex 5, so we add 5 to our array: (Tree[0] = 5) and remove vertex 1

  3. Repeat until 2 vertices are left

    • Remove vertex 2, add 6 to array: [5, 6]

    • Remove vertex 3, add 0 to array: [5, 6, 0]

    • Remove vertex 5, add 6 to array: [5, 6, 0, 6]

    • Remove vertex 6, add 0 to array: [5, 6, 0, 6, 0]

    • Remove vertex 0, add 4 to array: [5, 6, 0, 6, 0, 4]

    • Remove vertex 7, add 4 to array: [5, 6, 0, 6, 0, 4, 4]

    • Remove vertex 8, add 4 to array: [5, 6, 0, 6, 0, 4, 4, 4]

    • Stop conversion

In our case the array will look like this: [5, 6, 0, 6, 0, 4, 4, 4], and the tree will have just 2 connected vertices: 4 and 9
